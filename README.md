# 🏘️ 우리반 마을 (Our Class Village)

학급 단위의 가상 경제 시스템을 아기자기한 2D 픽셀아트 디자인과 융합한 **학급 경제 게임 웹 애플리케이션**입니다. 
학생들은 자신만의 2D 아바타로 마을을 돌아다니며 자원을 수확하고, 마켓에서 가구를 구매하여 방을 꾸미고, 학급 공공 직업(공무원) 활동을 수행하여 보상을 받거나 세금을 납부할 수 있습니다.

---

## 🎨 주요 특징 및 스택
- **프론트엔드**: Vanilla HTML, CSS, JavaScript (프레임워크 없음).
- **그래픽 엔진**: HTML5 Canvas를 이용한 2D 탑다운 월드 및 가구 인테리어 방 편집 시스템.
- **백엔드/DB**: Google Apps Script (GAS) `doGet` API + Google Sheets를 DB로 사용.
- **아키텍처**: 완전한 단일 페이지 애플리케이션(SPA) 구조로 페이지 전환 및 상호작용이 매끄럽게 흐릅니다.
- **오프라인 데모 폴백**: `js/config.js`에 GAS URL을 등록하기 전이라도, 브라우저 로컬 저장소(`localStorage`) 기반 시뮬레이터가 즉각 작동하여 **무설치 오프라인 플레이** 및 기능 확인이 가능합니다.

---

## 🕹️ 주요 기능

1. **마을 맵 탐색 (2D 탑다운 뷰)**
   - PC 키보드(WASD, 방향키) 및 모바일 반응형 가상 조이스틱 터치 지원.
   - 마을회관, 마켓, 경찰서, 세무서, 주택가 맵 포탈 구현.
   - 실시간 쿨다운 타이머가 내장된 수확 노드(사과나무, 꽃밭, 광석 바위).
   - 마을에 돌아다니는 다른 급우들의 아바타 NPC 대화 및 친구 집 방문 포탈.

2. **내 방 꾸미기 (2D 그리드 가구 배치)**
   - 10x8 격자 기반의 인테리어 코디네이팅.
   - 마켓에서 구매한 가구(침대, 책상, 의자, 책장 등) 배치 및 회수 지원.
   - 벽지 및 바닥 타일/카페트 실시간 테마 커스터마이징.
   - 방명록/한줄알림 기능 탑재 (주인 직접 작성 및 방문객 방명록 남기기 지원).

3. **마켓 상점**
   - 카테고리 필터링(가구, 벽지, 바닥지, 수확물)이 포함된 상품 목록.
   - 마켓운영자(또는 교사)를 위한 전역 물품 단가 조정, 재고 충전, 신규 가구 등록 대시보드.

4. **미션 게시판**
   - 교사가 직접 출제한 교실 돌발 과제 목록 열람.
   - 자기 보고 및 확인식 완료 요청을 통해 보상 코인 획득 및 내역 거래 원장 기록.

5. **마을 소식통 (공지사항)**
   - 학급 전체 중요 이벤트 및 법령 공지.
   - 중요 공지 지정 시, 로그인 후 첫 진입 시 모달 알림창 자동 팝업.

6. **직업 행정 통합 콘솔 (공무원 전용)**
   - **월급담당관**: 주민 체크박스 멀티 선택을 통해 일괄 급여 송금 지급.
   - **국세청장**: 소득세 및 주민세 등 일괄 세금 원천 징수 (잔액 초과 징수 방지 락 처리).
   - **경찰**: 학급 규칙 위반 주민 단일 선택으로 벌금 딱지 즉시 발부.
   - **교사 (최고관리자)**: 신규 학생 입주 등록(초기 집 자동 분양), 주민 직업 역할 임명, 학급 미션 하달, 전체 학생 개별 금융 감사(최근 50개 거래 장부 원장 추적).

## 🚀 빠른 시작 가이드 (배포 방법)

### 1단계: Google Apps Script 백엔드 배포
1. 구글 드라이브에서 [Google 스프레드시트]를 하나 생성합니다. (임의 이름 지정)
2. 스프레드시트 상단 메뉴의 **[확장 프로그램] > [Apps Script]**를 클릭합니다.
3. 이 저장소의 [gas/Code.gs](file:///Users/parkchangsoon/Downloads/안티그래비티/동물의숲/gas/Code.gs) 파일 내용을 복사하여 Apps Script 에디터에 붙여넣습니다.
4. 상단 메뉴의 **[배포] > [새 배포]**를 선택합니다.
5. 배포 유형을 **[웹 앱]**으로 선택하고 아래와 같이 설정합니다:
   - **설명**: `우리반마을 API`
   - **웹 앱을 실행할 사용자**: `나 (본인 구글 계정)`
   - **액세스할 수 있는 사용자**: `모든 사용자 (Anyone)`
6. **[배포]** 버튼을 누르고 구글 계정 권한 승인을 완료한 뒤, 발급된 **웹 앱 URL**(끝이 `/exec`로 끝남)을 복사합니다.

### 2단계: 프론트엔드 설정
1. 이 프로젝트의 `js/config.js` 파일을 엽니다.
2. `GAS_URL: ""` 부분의 큰따옴표 안에 복사한 웹 앱 URL을 입력하고 저장합니다.
   ```javascript
   const CONFIG = {
     GAS_URL: "https://script.google.com/macros/s/YOUR_DEPLOY_ID/exec",
     ...
   };
   ```

### 3단계: 초기 데이터베이스 구축 (최초 1회만)
1. 브라우저 주소창에 아래와 같이 입력하여 호출하거나, 웹앱 최초 로그인 후 화면에서 데이터베이스를 자동 빌드합니다.
   - 웹 앱 URL 뒤에 `?action=initializeDatabase`를 붙여 주소창에 입력하고 접속합니다.
   - 예: `https://script.google.com/macros/s/.../exec?action=initializeDatabase`
2. 구글 시트에 10개의 탭(`Users`, `Houses`, `MarketItems` 등)이 자동 생성되고 기초 물품 데이터와 기본 선생님 계정이 인입되면 완료됩니다.

### 4단계: GitHub Pages 또는 웹 호스팅 배포
1. 본인 GitHub 계정으로 저장소를 Push 또는 Fork 합니다.
2. 저장소의 **Settings > Pages** 메뉴로 이동합니다.
3. Build and deployment > Source를 `Deploy from a branch`로 설정하고 `main` 브랜치 `/ (root)` 폴더를 선택 후 **Save**를 누릅니다.
4. 잠시 후 완성되는 `https://<user-name>.github.io/<repo-name>/` 주소로 접속하면 학급 전원이 접속하여 실시간 연동되는 학급 경제 월드를 이용할 수 있습니다.

## 🧪 로컬 테스트 (오프라인 모드)
서버를 구성하지 않고 테스트하고 싶으시다면, `js/config.js`의 `GAS_URL`을 빈칸 `""`으로 둔 뒤 `index.html` 파일을 크롬 등 브라우저에 직접 드래그 앤 드롭으로 열기만 하세요! 

즉시 **브라우저의 독립 저장공간(localStorage)**에 테스트용 시뮬레이션 데이터베이스가 구축되며 로그인, 2D 맵 아바타 이동, 수확, 마켓 구매, 인테리어 룸 꾸미기, 공무원 권한 처리 등 **실제 백엔드와 100% 동일한 기능**을 즉석에서 검증하실 수 있습니다.
- **학생 계정**: `student1` (비밀번호: `1234`), `student2` (비밀번호: `1234`)
- **교사 계정**: `teacher` (비밀번호: `0000`)
